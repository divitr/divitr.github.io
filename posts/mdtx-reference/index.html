A comprehensive guide to the MDTX format syntax and features

Introduction

This document serves as a comprehensive reference for the MDTX format used in this blog. MDTX is a custom markdown format that extends standard markdown with LaTeX math support, custom blocks, and enhanced formatting options.

Basic Structure

Metadata Block

Every MDTX file begins with a metadata block that defines the document properties:

``` title: Document Title; date: Aug. 23, 2025; emoji: 🔬; visibility: visible; ```

Requirements (req:): Specifies LaTeX packages to load (e.g., physics, math) Title (title:): The main title of the document Date (date:): Publication date Description (desc:): Brief description for the blog index Tags (tags:): Comma-separated tags for categorization Emoji (emoji:): Optional emoji icon for the post Visibility (visibility:): Set to hidden to exclude from blog index (default: visible)

Theorem-like Environments

You can create theorem, lemma, proof, and other similar blocks with automatic numbering and cross-referencing.

Syntax

```mdtx [id][name]: ... content ... end ; ```

- <environment> can be theorem, lemma, definition, corollary, proposition, claim, remark, or proof. - Auto-numbering: All environments (except proof) share a sequential numbering: Theorem 1, Lemma 2, Definition 3, etc. - [id] (optional): A unique identifier for cross-referencing (e.g., central-limit) - [name] (optional): A display name shown in parentheses (e.g., Central Limit Theorem) - Syntax variations: - theorem: - just auto-numbered (e.g., "Theorem 1") -

Theorem 1 (My Theorem]: - numbered with ID and display name

Theorem Examples

Simple auto-numbered theorem: ```mdtx theorem: For any integer aaa and any prime ppp, we have ap≡a(modp)a^p \equiv a \pmod{p}ap≡a(modp). end theorem; ``` Renders as: Theorem 1. For any integer... Theorem with ID and name: ```mdtx theorem[fermat-little][Fermat's Little Theorem). For any integer aaa and any prime ppp, we have ap≡a(modp)a^p \equiv a \pmod{p}ap≡a(modp).

``` Renders as: Theorem 2 (Fermat's Little Theorem). For any integer...

Proof Environment

The proof environment has special features: - Visual styling: Subtle left border to distinguish proof blocks - QED symbol: Clickable □ box at the bottom right - Collapsible: Click the QED box to collapse/expand the proof - Expanded: Content visible, empty QED box (□) at bottom right - Collapsed: Content hidden, filled QED box (■) at top right - Reference linking: Can reference what theorem/lemma it proves

Cross-Reference Abbreviations

The abbrev@id syntax works in two places:

1. Proof headings — links the proof title to the environment it proves: ```mdtx

Proof (thm@my-theorem).

...

```

2. Inline in body text — renders as a hyperlinked "Type N" anywhere in prose: ```mdtx By lem@pair-count-lb, we have P(D)≥4P(D) \geq 4P(D)≥4. ``` Renders as: By Lemma 3, we have...

Supported abbreviations (short or full name both work): - thm@id / theorem@id - theorem - lem@id / lemma@id - lemma - def@id / definition@id - definition - cor@id / corollary@id - corollary - prop@id / proposition@id - proposition - claim@id - claim - rem@id / remark@id - remark

If the id doesn't exist or the abbreviation doesn't match the actual environment type, the text is left unchanged.

```mdtx

Lemma 3 (Helper Lemma). If x>0x > 0x>0, then x2>0x^2 > 0x2>0.

By Lemma 6, we know x2>0x^2 > 0x2>0. Multiplying both sides by x>0x > 0x>0...

```

This produces: - Lemma 1 (Helper Lemma). If x > 0... - Theorem 2 (Main Theorem). For all positive x... - Proof of Theorem 2. By Lemma 1...

Lists

Unordered Lists

Use list[u]: for unordered lists (bullet points):

  • First item
  • Second item with bold text
  • Third item with inlinemathinline mathinlinemath
  • Fourth item spanning multiple lines with indented continuation ### Ordered Lists Use list[o]: for ordered lists (numbered): list[o]: 1. First numbered item 2. Second item with emphasis 3. Third item with mathematicalcontentmathematical contentmathematicalcontent ### Alpha-Ordered Lists Use list[o-a]: for alphabetical ordering: list[o-a]: a. First item (alpha) b. Second item (beta) c. Third item (gamma) Note: Lists must end with end list; for best results, though in example boxes the end marker can be omitted and the list will end at the next unindented content. Nested lists are supported using indentation. ## Tables Tables use a LaTeX-inspired column spec and pipe-separated cells. ### Syntax ``` table[col-spec][Optional Caption]: Header 1 | Header 2 | Header 3 --- Row A | Row B | Row C end table; ``` Column spec [col-spec]: a string of l, c, r (alignment) and | (column borders): list[u]:
  • l — left-align
  • c — center-align
  • r — right-align
  • | between columns — draw a vertical line between them
  • Leading | — left outer border; trailing | — right outer border

Caption [Optional Caption]: shown below the table in italics. Omit the bracket entirely to skip it.

Header separator ---: the first --- marks the end of the header row(s). Rows above it become <thead> cells. If no --- appears, the table has no header.

Row separators ---: any additional --- after the header separator draws a horizontal line above the next row.

Compilation Process

The MDTX compiler processes files in the following order:

1. URL extraction - Extract all links and replace with placeholders 2. Comment removal - Remove lines starting with // 3. Metadata extraction - Parse title, date, description, tags, emoji, visibility 4. Requirements processing - Load specified LaTeX packages 5. Footnote extraction - Parse footnote definitions 6. Emphasis & inline code - Process bold, italic, and code 7. Display math - Convert block to LaTeX equation environments 8. Block elements - Process code blocks, images, examples, environments, lists, headings 9. Footnote references - Replace footnote markers with links 10. Paragraphs - Wrap text in paragraph tags 11. URL restoration - Restore links as HTML with target="_blank" 12. Inline math - Convert inline to LaTeX

Best Practices

List Usage

- Always use end list; for standalone lists - Lists in example boxes don't need explicit end markers - Use appropriate list types: [u] for unordered, [o] for ordered

Math Formatting

- Use ......... for inline math - Use block notation for display math - Leverage the physics package for common commands

Content Organization

- Use example boxes for important concepts - Structure content with clear headings - Include relevant images and code examples - Use footnotes for citations and references

Troubleshooting

Common Issues

1. Lists not rendering: Ensure you have end list; for standalone lists 2. Math not displaying: Check that LaTeX syntax is correct 3. Images not showing: Verify file paths and image files exist 4. Compilation errors: Check for syntax errors in metadata or content

Debug Tips

- Use the source link in the compiled HTML to view the original MDTX - Check the console for JavaScript errors - Verify that all referenced files exist in the correct locations

Conclusion

This reference covers all the major features of the MDTX format. The format is designed to be intuitive for users familiar with markdown while providing powerful extensions for mathematical content and structured presentation.
