A Category Theoretic View of Machine Learning

Using symmetries to better understand how to design attention kernels

Overview

Deep learning models often bake in symmetry and locality through ad-hoc tricks (e.g., convolutional tying, RoPE embeddings, SE(3)\text{SE(3)}SE(3)1 kernels, etc.). Our goal is to see if we can derive attention kernels from a declared symmetry and geometry, rather than hand-designing it.

Preliminaries

Categories

Category
A category C\mathcal CC composes objects. It consists of
  • Objects (the "things"), e.g., sets, vector spaces, etc., and
  • Morphisms f:X→Yf: X \to Yf:X→Y, the maps between objects,
so that we respect composition, i.e., g∘f:X→Zg \circ f: X \to Zg∘f:X→Z for f:X→Y,g:Y→Zf: X \to Y, g: Y \to Zf:X→Y,g:Y→Z, and we have an identity morphism 1X:X→X1_X: X\to X1X​:X→X for each object. Composition is associative (h∘(g∘f)=(h∘g)∘fh \circ (g \circ f) = (h \circ g) \circ fh∘(g∘f)=(h∘g)∘f), and the identity acts trivially (f∘1X=f=1Y∘ff \circ 1_X = f = 1_Y \circ ff∘1X​=f=1Y​∘f)

Categories are particularly useful because they give us (i) a nice way to think about symmetry, and (ii) a way to "translate" problems from one domain to another via functors.

Functors

Functor
A functor F:C→DF: \mathcal C \to \mathcal DF:C→D maps objects and morphisms while preserving composition: F(f:X→Y)=F(f):F(X)→F(Y),F(g∘f)=F(g)∘F(f) F(f: X \to Y) = F(f): F(X) \to F(Y),\quad F(g \circ f) = F(g) \circ F(f) F(f:X→Y)=F(f):F(X)→F(Y),F(g∘f)=F(g)∘F(f)

A functor acts as an embedding that maps geometric inputs into feature spaces.

If this embedding respects symmetries (e.g. rotation, permutation), we call it an equivariant functor: F(g⋅x)=ρ(g)F(x), F(g \cdot x) = \rho(g)F(x), F(g⋅x)=ρ(g)F(x), where ρ(g)\rho(g)ρ(g) is a tensor representation of the group action on outputs.

Natural Transformations

Two functors F,G:C→DF, G: \mathcal C \to \mathcal DF,G:C→D are related by a natural transformation ηX:F⇒G\eta_X: F \Rightarrow GηX​:F⇒G if for every object X, we have that ηX:F(X)→G(X) \eta_X: F(X) \to G(X) ηX​:F(X)→G(X) so that for every morphism f:X→Yf: X \to Yf:X→Y, G(f)∘ηX=ηY∘F(f). G(f) \circ \eta_X = \eta_Y \circ F(f). G(f)∘ηX​=ηY​∘F(f).

This is effectively equivalent to saying that changing the representation then applying a map is the same as applying a map then changing representation.

Ends

End
An end is a universal construction that represents the space of maps invariant under a family of morphisms. Formally, given a functor F:Cop×C→Set, F: \mathcal{C}^{\rm{op}} \times \mathcal{C} \to \mathbf{Set}, F:Cop×C→Set, the end ∫C∈CF(C,C) \int_{C \in \mathcal{C}} F(C, C) ∫C∈C​F(C,C) is the equalizer of all pairs of maps F(f,1)F(f, 1)F(f,1) and F(1,f)F(1, f)F(1,f) for every morphism fff in C\mathcal{C}C. Intuitively, it is the space of "globally consistent" elements -- things that remain unchanged as we move along all morphisms.

Intuitively, ends enforce symmetry consistency.

Coends

Coend
A coend is the dual notion of an end: instead of enforcing invariance, it glues local data into a consistent global object. Formally, for a functor F:Cop×C→Set, F: \mathcal{C}^{\mathrm{op}} \times \mathcal{C} \to \mathbf{Set}, F:Cop×C→Set, the coend ∫C∈CF(C,C) \int^{C \in \mathcal{C}} F(C, C) ∫C∈CF(C,C) is the coequalizer that identifies elements across overlaps induced by morphisms in C\mathcal{C}C.2

Intuitively, coends enforce spatial consistency.

Fibrations

A fibration is a morphism of categories π:E→B\pi: \mathcal{E} \to \mathcal{B}π:E→B that organizes "fibers" of structured objects over a base space. Intuitively, it tells us how a family of parameter spaces (the total category E\mathcal{E}E) projects onto model configurations (the base B\mathcal{B}B).

Putting it All Together

Equipped with the basic tools (categories, functors, natural transformations, and (co)ends), we are now ready to begin using these tools to better understand machine learning itself through the lens of category theory.

The key theme of the following is more or less that learning is composition inside a structured category.

Computation as a Category

A deep network is a morphism in a computational category C\mathcal CC: fθ:X→Y f_\theta: X \to Y fθ​:X→Y where objects are data/feature spaces and morphisms are differentiable3 maps parameterized by θ\thetaθ.

Then,

  • composition corresponds to stacking layers (h∘g)(x)=h(g(x))(h \circ g) (x) = h(g(x))(h∘g)(x)=h(g(x)),
  • identity corresponds to the skip connection,
  • and associativity guarantees that we may regroup computations arbitrarily (the chain rule is associative composition).

Then, because differentiation (backprop) is itself a functor RD:C→CRD: \mathcal C \to \mathcal CRD:C→C, the category is closed under gradients. Thus, learning is not some new addition to the model -- it is a morphism living in the same category.

Symmetry and Equivariance as Functorial Structure

A symmetry group GGG acting on data XXX induces a category of representations, where morphisms represent the group action.

An equivariant layer is then just a functor that preserves the action: F:Rep(G,X)→Rep(G,Y) F: \rm{Rep}(G,X) \to \rm{Rep}(G,Y) F:Rep(G,X)→Rep(G,Y) so that F(g⋅x)=ρY(g)F(x). F(g \cdot x) = \rho_Y(g)F(x). F(g⋅x)=ρY​(g)F(x).

Thus, CNNs, SE(3)SE(3)SE(3) Transformers and permutation-invariant networks are all functors that commute with their respective group actions. From this category theoretic view, we have the guarantee that a composition of equivariant functors is equivariant, so that equivariance is closed under composition.

Universality

Every architecture embeds assumptions about what should remain the same (invariance) and what should fit together (locality). Category theory expresses these as universal constructions:

  • The End ∫CF(C,C)\int_C F(C,C)∫C​F(C,C) is the “maximally consistent” object under all morphisms -- the space of global invariants.
  • The Coend ∫CF(C,C)\int^C F(C,C)∫CF(C,C) is the “minimal gluing” that merges overlapping structures.
Thus, together, Ends and Coends provide the principles that organize geometry and locality in learning systems.

Learning Dynamics as Fibration

Training is more than just optimization in parameter space -- it is a geometric process.

We may think of parameters M\mathcal MM as fibers sitting over models Θ\ThetaΘ via a fibration π:M→Θ. \pi: \mathcal M \to \Theta. π:M→Θ.

Each point in M\mathcal MM (a specific parameterization) correpsonds to the same function fθf_\thetafθ​ in Θ\ThetaΘ. A good learning rule is then one that moves "horizontally" along this bundle so that it changes the function itself, not just the coordinates.

A cartesian (natural) update T∘ϕ=ϕ∘T T \circ \phi = \phi \circ T T∘ϕ=ϕ∘T commutes with any reparametrization ϕ\phiϕ, meaning it is coordinate-free. This is the fundamental principle behind natural gradient and K-FAC methods.

Gauge Theories

The categorical framework we just built is near-identical to gauge theories in physics.

Fields, Bundles, and Equivariance

A field in physics assigns a quantity (e.g., a vector or tensor) to every point in space, subject to transformation laws under symmetries. Mathematically, this is a functor: F:(Spacetime)→(Fields) F: \text{(Spacetime)} \to \text{(Fields)} F:(Spacetime)→(Fields) that maps each region to its field content and each coordinate transformation to the corresponding tensor transformation.

Equivariance in ML plays the same role: F(g⋅x)=ρ(g)F(x), F(g \cdot x) = \rho(g) F(x), F(g⋅x)=ρ(g)F(x), where ggg is a symmetry (rotation, translation, permutation) and ρ(g)\rho(g)ρ(g) is its representation on the output space. An equivariant network is then a discrete field theory: a functor assigning representations to points on the data manifold.

Ends as Gauge Constraints

In a gauge theory, physical quantities are those that remain invariant under local transformations. An end ∫g∈GF(g,g) \int_{g \in G} F(g,g) ∫g∈G​F(g,g) plays exactly this role: it collects all quantities consistent under the action of every g∈Gg \in Gg∈G. In our framework, the end defines the space of legal attention kernels -- those consistent with the declared symmetry. This is the categorical version of a gauge constraint.

Coends as Bundle Gluing

A gauge field is not defined globally but patched together from local charts related by transition functions. The coend ∫Ui∩UjF(Ui,Uj) \int^{U_i \cap U_j} F(U_i, U_j) ∫Ui​∩Uj​F(Ui​,Uj​) formalizes this gluing. This ensures smooth global behavior, much like a well-constructed vector bundle or gauge potential.

Fibration as Connection

In gauge theory, a connection specifies how to move (parallel transport) along a fiber bundle without leaving the gauge orbit. In our framework, a fibration π:M→Θ \pi: \mathcal M \to \Theta π:M→Θ plays the same role: it organizes parameter space (M\mathcal MM) over model space (Θ\ThetaΘ), and the cartesian update T∘ϕ=ϕ∘T T \circ \phi = \phi \circ T T∘ϕ=ϕ∘T ensures that learning follows a connection that is natural -- invariant to reparameterization, just as covariant derivatives respect the gauge connection.

Deriving Known Kernels

This is cool and all, but without any application, it's not particularly useful. Here, we use the categorical language to derive some known results.

Namely, by recognizing that an end computes the space of all symmetry-consistent kernels, we may see that all equivariant attention mechanisms are instances of this same universal construction.

The End to Equivariant Kernels

Given a group GGG acting on an input manifold XXX and an output representation ρY\rho_YρY​ on YYY, the space of all admissible kernels is the end: KerG(X,Y)=∫g∈GC(g⋅(X×X), g⋅End(Y)). \mathsf{Ker}_G(X, Y) = \int_{g \in G} \mathcal{C}\left(g \cdot (X \times X),\, g \cdot \mathrm{End}(Y)\right). KerG​(X,Y)=∫g∈G​C(g⋅(X×X),g⋅End(Y)).

In other words, that means that KKK commutes with the symmetry action, the categorical analog of a gauge-invariant kernel.

Translations: Convolutions and CNN Tying

For G=R2G = \mathbb{R}^2G=R2 (translations on the image plane) acting by shifts g⋅x=x+gg \cdot x = x + gg⋅x=x+g, equivariance requires K(q+g,k+g)=K(q,k). K(q + g, k + g) = K(q, k). K(q+g,k+g)=K(q,k). Thus, K(q,k)=κ(k−q)K(q, k) = \kappa(k - q)K(q,k)=κ(k−q): the kernel depends only on relative position. This is exactly the weight-tying rule of convolution -- a convolution layer is simply an end over the translation group.

Rotations: RoPE and SO(2)SO(2)SO(2) Equivariant Attention

For G=SO(2)G = \mathrm{SO}(2)G=SO(2)4, the end yields kernels invariant under rotation: K(Rϕq,Rϕk)=RϕK(q,k)Rϕ−1. K(R_\phi q, R_\phi k) = R_\phi K(q, k) R_\phi^{-1}. K(Rϕ​q,Rϕ​k)=Rϕ​K(q,k)Rϕ−1​. Parameterizing KKK in polar coordinates gives K(q,k)=∑ℓ=0Lfℓ(r)[cos⁡(ℓϕ),sin⁡(ℓϕ)], K(q, k) = \sum_{\ell=0}^{L} f_\ell(r) [\cos(\ell \phi), \sin(\ell \phi)], K(q,k)=ℓ=0∑L​fℓ​(r)[cos(ℓϕ),sin(ℓϕ)], where r=∣k−q∣r = \left| k - q \right|r=∣k−q∣ and ϕ\phiϕ is the relative angle. This recovers rotational position embeddings (RoPE) and circular harmonics kernels as the unique solutions to the equivariance constraint.

3D Rotations: SE(3)SE(3)SE(3) and Molecular Attention

For G=SO(3)G = \mathrm{SO}(3)G=SO(3)5 or E(3)E(3)E(3)6, the end requires rotational (and possibly translational) invariance: K(gq,gk)=K(q,k). K(g q, g k) = K(q, k). K(gq,gk)=K(q,k). Expanding in spherical harmonics gives K(x,y)=∑ℓ=0LaℓPℓ(x⋅y), K(x, y) = \sum_{\ell=0}^L a_\ell P_\ell(x \cdot y), K(x,y)=ℓ=0∑L​aℓ​Pℓ​(x⋅y), where PℓP_\ellPℓ​ are Legendre polynomials or real spherical harmonics. These are the basis functions used in SE(3)SE(3)SE(3) Transformers and tensor field networks.

Permutations: DeepSets and Graph Attention

For G=SnG = S_nG=Sn​7, equivariance requires K(σq,σk)=K(q,k). K(\sigma q, \sigma k) = K(q, k). K(σq,σk)=K(q,k). The unique linear form is K=αI+β11⊤, K = \alpha I + \beta \mathbf{1}\mathbf{1}^\top, K=αI+β11⊤, which is exactly the kernel used in DeepSets and permutation-equivariant graph networks. If we further restrict interactions by adjacency, we recover message passing.

Conclusion

Thus, we have that

  • category theory gives the language (composition, functors, ends, coends, fibrations),
  • physics supplies the intuition (symmetries, gluing, connections),
  • machine learning is the instantiation (attention, patches, optimization).
At some point, it'd probably be cool to write up some code that computes legal kernels given the symmetries that your data respects, but that's a project for later.
